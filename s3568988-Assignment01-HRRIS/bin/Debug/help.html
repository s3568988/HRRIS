Expanded Use Case

  • Use case: Insert record in the booking table
  • Actor: Hotel staffs, managers
  • Purpose: Record booking details
  • Type: Abstract
  • Importance: Primary
  • Technical difficulty: medium
  • Cross references: none
  • Pre-condition: none
  • Assumption: There must be record in the customer and booking table
  • Typical course of event
    • Actor action Systems response
    1. User open booking form  
      2. Booking form is pop-up
    3. User fill in the form  
    4. User check the record
      5. System validate the each record
    6. User press Save button  
      7. Systems record details
  • Alternative course

    Line 5: problem in validation, systems validate incorrectly
